User Access Guide

A complete reference for Principals & Administrators — detailing what each role can see, what they cannot, and why that matters.

📘 14 roles · 32 features · 3 layers of security
ℹ️

The system defines 14 distinct user roles. Each role is granted access only to what it needs for its duties — a design decision rooted in staff privacy, student safety, and financial integrity. The admin sidebar exposes 32 features in total; roles see a curated subset of these.

🛡️ Three layers of system security

Layer 01

Database layer

PostgreSQL Row-Level Security — even bypassing the API cannot leak data.

Layer 02

Server layer

Every action passes through a requirePermission() check.

Layer 03

UI layer

Buttons and links are hidden from users who do not have permission to use them.

✓ Result: An accountant can never view or change exam marks — even by manipulating the URL.
